## Feature Overview

The Product Tagging UIKit components provide comprehensive tools for implementing product discovery and commerce features throughout your application. Users can tag products in posts and livestreams, viewers can browse tagged products, and hosts can pin featured products for maximum visibility. These components integrate seamlessly across multiple content types and support real-time synchronization.

## Product Tagging Types & Features

<AccordionGroup>
  <Accordion title="Post Product Tagging" icon="pen-to-square">
    **Product tagging in post creation and viewing**

    - **Text Product Mentions**: Tag products in post captions using `@` trigger with product suggestion overlay
    - **Media Product Tags**: Tag products directly on images and videos with coordinate-based positioning
    - **Product Selection UI**: Full-screen or bottom sheet product search and multi-select interface
    - **Limit Enforcement**: Maximum 20 product tags per post (combined text mentions and media tags)
    - **Post Editing**: Edit product tags on existing posts while maintaining engagement metrics
    - **Product Carousel**: Display all tagged products in a scrollable carousel in post detail and feed views
  </Accordion>

  <Accordion title="Livestream Product Tagging" icon="video">
    **Real-time product management during livestreams**

    - **Pre-Livestream Tagging**: Host can pre-tag products before broadcast begins
    - **Live Tagging**: Host and authorized co-hosts can tag products during active broadcast
    - **Post-Livestream Management**: Host can manage products after livestream ends for replay view
    - **Real-time Synchronization**: Product changes sync instantly to all participants and viewers
    - **Product Ordering**: Products appear in order added with persistent ordering maintained
    - **Product Limit**: Maximum 20 products per livestream enforced across all tagging phases
  </Accordion>

  <Accordion title="Product Pinning" icon="thumbtack">
    **Featured product overlay for viewer focus**

    - **Single Pin Limit**: Only one product can be pinned at a time
    - **Pinned Overlay Display**: Prominent product card showing image, name, price, and CTA
    - **Real-time Pin Updates**: Pin changes sync immediately to all viewers
    - **Pin Priority**: Pinned product always appears at top of product list
    - **Auto-unpin**: Pinned product automatically unpins if removed from tag list
    - **Replay Support**: Pin overlay removed when livestream ends, product list remains accessible
  </Accordion>

  <Accordion title="Co-host Product Permissions" icon="shield-check">
    **Control co-host access to product management features**

    - **Permission Toggle**: Host can enable/disable co-host product management capabilities
    - **Granular Control**: Co-hosts can tag, untag, pin, and unpin products when permissions enabled
    - **Real-time Permission Changes**: Permission changes apply immediately without requiring restart
    - **UI State Management**: Product controls shown/hidden based on permission status
    - **Backstage Configuration**: Permissions granted during backstage co-host acceptance
    - **Menu Access**: Permission toggles accessible from co-host name menu
  </Accordion>

  <Accordion title="Viewer Product Browsing" icon="shopping-cart">
    **Product discovery for livestream and post viewers**

    - **Product List Overlay**: Scrollable overlay displaying all tagged products with quick access
    - **Product Card Display**: Each product shows image, name, price, and call-to-action button
    - **Pinned Product Highlight**: Prominent display for featured product with special styling
    - **Dynamic Tag Counter**: Real-time badge showing number of tagged products
    - **Tag Icon Visibility**: Tag icon only appears when one or more products are tagged
    - **In-app Browser**: Open product links within app without interrupting streaming experience
  </Accordion>

  <Accordion title="Product Analytics" icon="chart-bar">
    **Track product engagement metrics**

    - **View Tracking**: Record when products become visible to users
    - **Click Tracking**: Track user interactions with product cards
    - **Pinned Product Metrics**: Separate analytics for featured product engagement
    - **Source Attribution**: Track whether interaction originated from pinned or list view
    - **Offline Support**: Queue analytics events when offline, sync on reconnection
    - **Deduplication**: Prevent duplicate view events per product per session
  </Accordion>
</AccordionGroup>

## Best Practices

<AccordionGroup>
  <Accordion title="Product Tagging Strategy" icon="tag">
    - **Relevance First**: Tag products that are directly relevant to the content
    - **Moderate Tagging**: Use 3-8 tags per post for optimal engagement (under the 20 limit)
    - **Timing**: Tag products early in content creation for better visibility
    - **Catalog Quality**: Ensure product catalog is up-to-date with accurate descriptions and images
    - **Pricing Accuracy**: Keep product prices synchronized for viewer trust
  </Accordion>

  <Accordion title="Livestream Product Management" icon="broadcast">
    - **Strategic Pinning**: Pin highest-value or trending products for maximum visibility - **Real-time Updates**: Update product list during stream based on viewer engagement - **Co-host Coordination**: Communicate product tagging strategy with authorized co-hosts - **Phase-based Strategy**: Plan different products for setup, live, and wrap-up phases - **Monitor Performance**: Track which products get most engagement during stream
  </Accordion>

  <Accordion title="User Experience" icon="heart">
    - **Clear Visual Feedback**: Show confirmation when products are tagged/untagged - **Intuitive Navigation**: Make product list easily accessible but non-intrusive - **Smooth Browsing**: Optimize product list scrolling for mobile devices - **In-app Navigation**: Always use in-app browser for product links to maintain context - **Accessibility**: Ensure product tags have proper ARIA labels and keyboard navigation
  </Accordion>

  <Accordion title="Performance Optimization" icon="gauge">
    - **Lazy Loading**: Load product list only when user interacts with tag icon - **Image Optimization**: Use appropriately sized product images to reduce load times - **Sync Efficiency**: Batch product tag updates to reduce network requests - **Caching**: Cache product catalog locally to minimize API calls - **Background Updates**: Handle real-time sync without blocking UI rendering
  </Accordion>

  <Accordion title="Analytics Integration" icon="chart-bar">
    - **Track Engagement**: Monitor product views and clicks from posts and livestreams
    - **Compare Sources**: Analyze performance difference between pinned and list products
    - **Session Tracking**: Correlate product interactions with overall session metrics
    - **Attribution**: Track conversions back to specific posts or livestreams
    - **A/B Testing**: Test different product tagging strategies and measure impact
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Product Tags Not Appearing" icon="eye-slash">
    **Problem**: Tagged products not visible in post or livestream

    **Solutions**:

    - Verify product tagging feature is enabled in console settings
    - Check that product catalog has been properly synchronized
    - Ensure user has appropriate permissions for tagging
    - Validate that products exist and are active in catalog
    - Clear app cache and reload content

    ```swift
    // Verify product tagging is enabled
    let config = AmityUIKitManager.getProductTaggingConfiguration()
    if config.isEnabled {
        // Product tagging should be available
    }
    ```
  </Accordion>

  <Accordion title="Real-time Sync Issues" icon="arrows-rotate">
    **Problem**: Product changes not syncing to other participants

    **Solutions**:

    - Check network connectivity for all participants
    - Verify websocket connection is active
    - Restart livestream if sync issues persist
    - Check server logs for sync failures
    - Ensure all participants have required permissions

    ```typescript
    // Monitor sync status
    const syncStatus = livestreamManager.getSyncStatus();
    if (!syncStatus.isConnected) {
      // Handle disconnection
      handleReconnection();
    }
    ```
  </Accordion>

  <Accordion title="Permission Issues" icon="lock">
    **Problem**: Co-host cannot access product tagging controls

    **Solutions**:

    - Verify co-host has been granted product management permission
    - Check permission settings in backstage controls
    - Ensure co-host role has been properly assigned
    - Restart backstage if permissions don't reflect
    - Verify host has permission to grant permissions to co-hosts

    ```kotlin
    // Check co-host permissions
    val hasPermission = livestreamManager.checkCoHostPermission(
        coHostId = coHostId,
        permission = "product_management"
    )
    ```
  </Accordion>

  <Accordion title="Product Catalog Issues" icon="database">
    **Problem**: Product search returns no results or incorrect products

    **Solutions**:

    - Verify product catalog is properly synchronized
    - Check product metadata is complete and searchable
    - Ensure products are marked as active/available
    - Validate search index is updated
    - Check integration with product management system
  </Accordion>

  <Accordion title="Image Loading" icon="image">
    **Problem**: Product images not displaying in carousel or overlay

    **Solutions**:

    - Verify image URLs are valid and accessible
    - Check image size optimization settings
    - Ensure CDN is properly configured
    - Clear image cache and reload
    - Validate image format support (JPEG, PNG, WebP)
  </Accordion>
</AccordionGroup>
