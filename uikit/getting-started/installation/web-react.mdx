## Package Installation

Perfect for quick integration with pre-built components and minimal setup.

### Prerequisites

Ensure you have the following installed:

<CardGroup cols={2}>
  <Card title="Node.js LTS" icon="node-js">
    Version 20 or higher
  </Card>
  <Card title="PNPM" icon="package">
    Version 8 or higher (recommended)
  </Card>
</CardGroup>

### Setup Steps

<Steps>
  <Step title="Install PNPM (Optional)">
    If you don't have PNPM installed, enable it using corepack:
    
    ```bash
    corepack enable pnpm
    ```
    
    [Learn more about PNPM installation â†’](https://pnpm.io/installation#using-corepack)
  </Step>
  
  <Step title="Install UIKit Package">
    Install the Social+ UIKit package via PNPM:
    
    ```bash
    pnpm i @amityco/ui-kit
    ```
    
    <Note>
      You can also use npm or yarn:
      - `npm install @amityco/ui-kit`
      - `yarn add @amityco/ui-kit`
    </Note>
  </Step>
  
  <Step title="Import Required Styles">
    Add the main UIKit styles to your application entry point:
    
    ```typescript
    import "@amityco/ui-kit-open-source/dist/index.css";
    ```
  </Step>
  
  <Step title="Setup Provider">
    Wrap your application with `AmityUiKitProvider`:
    
    ```typescript
    import React from 'react';
    import { AmityUiKitProvider, AmityUiKitSocial } from '@amityco/ui-kit-open-source';
    import "@amityco/ui-kit-open-source/dist/index.css";
    import amityConfig from './amity-uikit.config.json';
    
    export default function App() {
      return (
        <AmityUiKitProvider
          apiKey={apiKey}
          userId={userId}
          displayName={displayName}
          apiEndpoint={apiEndpoint}  // Optional if using default endpoints
          apiRegion={apiRegion} // "us", "eu", or "sg"
          configs={amityConfig as Config}
        >
          <div
            style={{
              position: "absolute",
              left: 0,
              top: 0,
              width: "100vw",
              height: "100dvh",
            }}
          >
            <AmityUiKitSocial />
          </div>
        </AmityUiKitProvider>
      );
    }
    ```
  </Step>
</Steps>

### Configuration

<Accordion title="Provider Configuration Options">
  | Property | Type | Required | Description |
  |----------|------|----------|-------------|
  | `apiKey` | string | Yes | Your Social+ API key |
  | `userId` | string | Yes | Unique identifier for the user |
  | `displayName` | string | Yes | User's display name |
  | `apiRegion` | string | Yes | API region: "us", "eu", or "sg" |
  | `apiEndpoint` | string | No | Custom API endpoint (optional) |
  | `configs` | Config | No | Custom theme and styling configuration |
</Accordion>

## Customization

### Theme Configuration

Create a configuration file `amity-uikit.config.json` to customize the UIKit appearance:

```json
{
  "global_theme": {
    "light_theme": {
      "primary_color": "#FFFFFF",
      "secondary_color": "#AB1234",
      "accent_color": "#1E88E5",
      "base_color": "#000000"
    },
    "dark_theme": {
      "primary_color": "#1A1A1A",
      "secondary_color": "#AB1234",
      "accent_color": "#64B5F6",
      "base_color": "#FFFFFF"
    }
  },
  "component_theme": {
    "social_feed": {
      "background_color": "#F5F5F5",
      "card_background": "#FFFFFF"
    },
    "chat": {
      "message_bubble_user": "#1E88E5",
      "message_bubble_other": "#E0E0E0"
    }
  }
}
```

<AccordionGroup>
  <Accordion title="Global Theme Options">
    - `primary_color`: Main brand color
    - `secondary_color`: Secondary brand color
    - `accent_color`: Accent highlights and CTAs
    - `base_color`: Text and base UI elements
  </Accordion>
  
  <Accordion title="Component Specific Themes">
    - `social_feed`: Feed layout and post styling
    - `chat`: Chat interface colors
    - `user_profile`: Profile page styling
    - `navigation`: Navigation bar appearance
  </Accordion>
</AccordionGroup>

## Open Source Installation

Choose open source for maximum flexibility, complete customization control, and development transparency.

### Benefits of Open Source

<CardGroup cols={3}>
  <Card title="Complete Control" icon="sliders">
    Full access to source code for unlimited customization
  </Card>
  <Card title="Transparency" icon="eye">
    Understand exactly how components work
  </Card>
  <Card title="Community" icon="users">
    Contribute and benefit from community improvements
  </Card>
</CardGroup>

### Development Setup

<Steps>
  <Step title="Clone Repository">
    Clone the open source UIKit repository:
    
    ```bash
    git clone https://github.com/AmityCo/Amity-Social-Cloud-UIKit-Web-OpenSource.git
    cd Amity-Social-Cloud-UIKit-Web-OpenSource
    ```
  </Step>
  
  <Step title="Install Dependencies">
    Install project dependencies:
    
    ```bash
    pnpm install
    ```
  </Step>
  
  <Step title="Environment Setup">
    Create a `.env` file in the project root:
    
    ```bash
    STORYBOOK_API_REGION=<API_REGION>
    STORYBOOK_API_KEY=<API_KEY>
    ```
    
    Replace `<API_REGION>` and `<API_KEY>` with your actual credentials.
  </Step>
  
  <Step title="Run Storybook (Optional)">
    View components in isolation using Storybook:
    
    ```bash
    pnpm run storybook
    ```
    
    Open `http://localhost:6006` in your browser.
  </Step>
</Steps>

### Integration with Your Project

<Steps>
  <Step title="Build the UIKit">
    Build the UIKit project:
    
    ```bash
    pnpm run build
    ```
  </Step>
  
  <Step title="Link to Your Application">
    Navigate to your application directory and link the UIKit:
    
    <Tabs>
      <Tab title="NPM">
        ```bash
        npm link file:<path-to-amity-ui-kit-repository> --save
        ```
      </Tab>
      <Tab title="Yarn">
        ```bash
        yarn add file:<path-to-amity-ui-kit-repository>
        ```
      </Tab>
      <Tab title="PNPM">
        ```bash
        pnpm i file:<path-to-amity-ui-kit-repository>
        ```
      </Tab>
    </Tabs>
  </Step>
  
  <Step title="Import and Use">
    Import and use the UIKit in your application:
    
    ```typescript
    import { AmityUiKitProvider, AmityUiKitSocial } from '@amityco/ui-kit-open-source';
    import "@amityco/ui-kit-open-source/dist/index.css";
    ```
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Module Resolution Issues">
    If you encounter module resolution problems:
    1. Clear node_modules: `rm -rf node_modules && pnpm install`
    2. Clear package manager cache: `pnpm store prune`
    3. Restart your development server
  </Accordion>
  
  <Accordion title="Style Import Issues">
    For CSS import problems:
    1. Ensure styles are imported in your main entry file
    2. Check if your bundler supports CSS imports
    3. Verify the import path: `@amityco/ui-kit-open-source/dist/index.css`
  </Accordion>
  
  <Accordion title="Build Errors">
    Common build error solutions:
    1. Verify Node.js version compatibility (LTS 20+)
    2. Check for conflicting dependencies
    3. Ensure all peer dependencies are installed
    4. Try clearing build cache and rebuilding
  </Accordion>
  
  <Accordion title="Provider Configuration">
    If the provider isn't working correctly:
    1. Verify all required props are provided
    2. Check API credentials and region settings
    3. Ensure the provider wraps all components that need Social+ context
    4. Validate the configuration JSON structure
  </Accordion>
</AccordionGroup>
