## SwiftPM Installation

Perfect for quick integration with minimal configuration. Use this method if you want to get started quickly with pre-built components.

### Prerequisites

<Steps>
  <Step title="Create iOS Project">
    Open Xcode and create a new iOS project:
    - Enter your project name
    - Select **Swift** as the language
    - Choose your deployment target (iOS 12.0+)
  </Step>
</Steps>

### Package Installation

<Steps>
  <Step title="Add Package Dependency">
    In Xcode, navigate to **File → Add Package Dependencies** and enter the repository URL:
    
    ```
    https://github.com/AmityCo/Amity-Social-Cloud-UIKit-iOS-SwiftPM
    ```
  </Step>
  
  <Step title="Select Version">
    <Warning>
      Use **Exact Version** dependency rule for UIKit v4 beta versions.
      
      Example: `Exact Version` → `4.6.0`
    </Warning>
    
    Choose `AmityUIKit` from the package products list.
  </Step>
  
  <Step title="Import and Use">
    After installation, import the framework in your Swift files:
    
    ```swift
    import AmityUIKit
    
    // Example usage
    let chatViewController = AmityChatViewController()
    navigationController?.pushViewController(chatViewController, animated: true)
    ```
  </Step>
</Steps>

<Tip>
  The SwiftPM installation automatically handles all dependencies and provides a streamlined integration experience.
</Tip>

## Open Source Installation

Choose open source for maximum flexibility, complete customization control, and community contributions.

### Benefits of Open Source

<AccordionGroup>
  <Accordion title="Complete Customization">
    Full control over UI components, styling, and behavior to match your brand perfectly.
  </Accordion>
  
  <Accordion title="Transparency">
    Access to complete source code for better understanding and debugging capabilities.
  </Accordion>
  
  <Accordion title="Community Contributions">
    Benefit from community improvements, bug fixes, and feature enhancements.
  </Accordion>
</AccordionGroup>

### Migration from Managed UIKit

<Warning>
  If you're currently using the managed UIKit, remove all existing dependencies before proceeding with open source installation.
</Warning>

### Installation Steps

<Steps>
  <Step title="Add Git Submodule">
    Add the open source UIKit as a git submodule to your repository:
    
    ```bash
    git submodule add https://github.com/AmityCo/ASC-UIKit-iOS-OpenSource
    ```
    
    <Note>
      **Repository Migration Notice**: We've moved to a new repository starting from v3.12.0. The new repository at `ASC-UIKit-iOS-OpenSource` is now the active development location.
    </Note>
  </Step>
  
  <Step title="Create Xcode Workspace">
    1. Create a new Xcode Workspace
    2. Add both your app project (`MyApp.xcodeproj`) and `AmityUIKit.xcodeproj` to the workspace
    
    Your workspace structure should look like:
    ```
    MyWorkspace.xcworkspace
    ├── MyApp.xcodeproj
    └── ASC-UIKit-iOS-OpenSource/
        └── AmityUIKit.xcodeproj
    ```
  </Step>
  
  <Step title="Link Dependencies">
    **Step 1**: Reset Package Cache (if upgrading from existing installation)
    - Select your app target in Xcode
    - Go to **File → Packages → Reset Package Cache**
    
    **Step 2**: Link Required Frameworks
    - Add `SharedFrameworks` (Swift Package dependency)
    - Add `AmityUIKit.framework` to your app target
    
    Both should be added to your app target's **Frameworks, Libraries, and Embedded Content** section.
  </Step>
  
  <Step title="Import and Test">
    Test the integration by importing the framework:
    
    ```swift
    import AmityUIKit
    import AmitySDK
    
    // Test the integration
    class ViewController: UIViewController {
        override func viewDidLoad() {
            super.viewDidLoad()
            // Your Social+ UIKit implementation
        }
    }
    ```
  </Step>
</Steps>

### Repository Migration Guide

<AccordionGroup>
  <Accordion title="For New Users">
    Start with the new repository: `https://github.com/AmityCo/ASC-UIKit-iOS-OpenSource`
    
    If you need to fork the repository, use the new one for all future development.
  </Accordion>
  
  <Accordion title="For Existing Users (< v3.12.0)">
    1. **Upgrade to v3.12.0** from the old repository first
    2. **Migrate to new repository** once you're on v3.12.0+

  </Accordion>
</AccordionGroup>

### Customization

<Steps>
  <Step title="Fork the Repository">
    Fork the repository to maintain your customizations while staying up-to-date with official updates:
    
    ```bash
    # Clone your fork
    git clone https://github.com/YOUR_USERNAME/ASC-UIKit-iOS-OpenSource
    ```
    
    [Learn more about forking →](https://docs.github.com/en/get-started/quickstart/fork-a-repo)
  </Step>
  
  <Step title="Make Customizations">
    Modify the source code directly to fit your needs:
    - Update UI components
    - Change color schemes and styling
    - Add custom behaviors
    - Implement brand-specific features
  </Step>
  
  <Step title="Stay Updated">
    Regularly sync with the main repository to get latest updates:
    
    ```bash
    # Add upstream remote
    git remote add upstream https://github.com/AmityCo/ASC-UIKit-iOS-OpenSource
    
    # Fetch and merge updates
    git fetch upstream
    git merge upstream/main
    ```
  </Step>
</Steps>

### Update to Latest Version

Keep your open source UIKit updated with the latest features and fixes:

```bash
# Navigate to the submodule directory
cd ASC-UIKit-iOS-OpenSource

# Pull latest changes
git pull origin master

# Checkout specific version (recommended)
git checkout tags/4.0.0-beta01

# Return to parent directory and commit changes
cd ..
git add .
git commit -m "Update iOS UIKit open source submodule to v4.0.0-beta01"
git push origin main
```

<Tip>
  Always test thoroughly after updating to ensure compatibility with your customizations.
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Package Resolution Issues">
    If you encounter package resolution problems:
    1. Clean build folder: **Product → Clean Build Folder**
    2. Reset package cache: **File → Packages → Reset Package Cache**
    3. Update package versions: **File → Packages → Update to Latest Package Versions**
  </Accordion>
  
  <Accordion title="Linking Errors">
    For framework linking issues:
    1. Verify both `SharedFrameworks` and `AmityUIKit.framework` are properly linked
    2. Check that your deployment target meets minimum requirements (iOS 12.0+)
    3. Ensure you're using the correct Xcode workspace, not individual projects
  </Accordion>
  
  <Accordion title="Build Errors">
    Common build error solutions:
    1. Update to latest Xcode version
    2. Verify Swift version compatibility
    3. Check for conflicting dependencies
    4. Clean derived data: **Xcode → Preferences → Locations → Derived Data → Delete**
  </Accordion>
</AccordionGroup>
