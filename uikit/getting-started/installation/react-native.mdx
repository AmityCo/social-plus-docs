## Open Source Installation

Perfect for maximum customization and development flexibility.

### Build Process

<Steps>
  <Step title="Clone Repository">
    Clone the Social+ UIKit open-source repository:
    
    ```bash
    git clone https://github.com/AmityCo/Amity-Social-UIKit-React-Native-CLI-OpenSource.git
    cd Amity-Social-Cloud-UIKit-React-Native-OpenSource
    ```
  </Step>
  
  <Step title="Install Dependencies">
    Install the required dependencies:
    
    ```bash
    yarn install
    ```
  </Step>
  
  <Step title="Build UIKit">
    Build the UIKit package:
    
    ```bash
    npm run prepack
    npm pack
    ```
    
    This creates `amity-react-native-social-ui-kit-x.x.x.tgz` in the project directory.
  </Step>
</Steps>

## Platform Configuration

### iOS Configuration

<Steps>
  <Step title="Update Podfile">
    Add these lines to your `ios/Podfile` under your target:
    
    ```ruby
    pod 'SPTPersistentCache', :modular_headers => true
    pod 'DVAssetLoaderDelegate', :modular_headers => true
    $RNVideoUseVideoCaching = true
    ```
  </Step>
  
  <Step title="Set Deployment Target">
    In Xcode, set the **Minimum Deployment Target** to **iOS 12.0** or higher.
  </Step>
  
  <Step title="Install Pods">
    Install the iOS dependencies:
    
    ```bash
    cd ios
    pod install
    cd ..
    ```
  </Step>
</Steps>

### iOS Permissions

Add the following permissions to `ios/{YourAppName}/Info.plist`:

```xml
<key>NSCameraUsageDescription</key>
<string>App needs access to the camera to take photos.</string>
<key>NSMicrophoneUsageDescription</key>
<string>App needs access to the microphone to record audio.</string>
<key>NSPhotoLibraryUsageDescription</key>
<string>App needs access to the gallery to select photos.</string>
```

### iOS Background Downloads

Add this method to `ios/{YourAppName}/AppDelegate.mm`:

```objc
#import <RNFSManager.h>

// Add this method to your AppDelegate implementation
- (void)application:(UIApplication *)application 
handleEventsForBackgroundURLSession:(NSString *)identifier 
completionHandler:(void (^)())completionHandler
{
  [RNFSManager setCompletionHandlerForIdentifier:identifier 
                                 completionHandler:completionHandler];
}
```

### Android Configuration

<Steps>
  <Step title="Update Build Configuration">
    In `android/build.gradle`, update the buildscript > ext section:
    
    ```gradle
    buildscript {
        ext {
            kotlinVersion = "1.7.0" // or higher
            compileSdkVersion = 34
            buildToolsVersion = "34.0.0"
            // ... other configurations
        }
    }
    ```
  </Step>
  
  <Step title="Add Permissions">
    Add the following permissions to `android/app/src/main/AndroidManifest.xml`:
    
    ```xml
    <uses-permission android:name="android.permission.CAMERA" />
    <uses-permission android:name="android.permission.RECORD_AUDIO" />
    ```
  </Step>
</Steps>

## Basic Usage

### Provider Setup

Wrap your app with the `AmityUiKitProvider`:

```typescript
import React from 'react';
import {
  AmityUiKitSocial,
  AmityUiKitProvider,
} from 'amity-react-native-social-ui-kit';

export default function App() {
  return (
    <AmityUiKitProvider
      apiKey="YOUR_API_KEY"
      apiRegion="YOUR_API_REGION" // "us", "eu", or "sg"
      userId="USER_ID"
      displayName="USER_DISPLAY_NAME"
      apiEndpoint="https://api.{API_REGION}.amity.co"
    >
      <AmityUiKitSocial />
    </AmityUiKitProvider>
  );
}
```

### Configuration Options

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `apiKey` | string | Yes | Your Social+ API key |
| `apiRegion` | string | Yes | API region: "us", "eu", or "sg" |
| `userId` | string | Yes | Unique identifier for the user |
| `displayName` | string | Yes | User's display name |
| `apiEndpoint` | string | Yes | API endpoint URL |
| `configs` | object | No | Custom theme and configuration |

## Customization

### Theme Configuration

Create a `uikit.config.json` file in your project root for customization:

```typescript
import React from 'react';
import {
  AmityUiKitSocial,
  AmityUiKitProvider,
} from 'amity-react-native-social-ui-kit';
import config from './uikit.config.json';

export default function App() {
  return (
    <AmityUiKitProvider
      configs={config}
      apiKey="YOUR_API_KEY"
      apiRegion="YOUR_API_REGION"
      userId="USER_ID"
      displayName="USER_DISPLAY_NAME"
      apiEndpoint="https://api.{API_REGION}.amity.co"
    >
      <AmityUiKitSocial />
    </AmityUiKitProvider>
  );
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Build Issues">
    **Android Build Problems:**
    1. Ensure Kotlin version is 1.7.0 or higher
    2. Check that compileSdkVersion is 34+
    3. Verify NDK version is 25.1.8937393+
    4. Clean and rebuild: `cd android && ./gradlew clean && cd .. && npx react-native run-android`
    
    **iOS Build Problems:**
    1. Clean build folder in Xcode
    2. Delete `ios/Pods` and `ios/Podfile.lock`, then run `pod install`
    3. Ensure minimum deployment target is iOS 13.4+
  </Accordion>
  
  <Accordion title="Dependency Conflicts">
    If you encounter peer dependency issues:
    1. Check React Native version compatibility
    2. Ensure all peer dependencies are installed with correct versions
    3. Try using `--force` flag with npm/yarn if needed
    4. Clear package manager cache and reinstall
  </Accordion>
  
  <Accordion title="Runtime Errors">
    **Metro bundler issues:**
    1. Clear Metro cache: `npx react-native start --reset-cache`
    2. Ensure all native dependencies are properly linked
    3. Check that permissions are properly configured for each platform
    
    **Native module issues:**
    1. For iOS: Run `cd ios && pod install`
    2. For Android: Clean and rebuild the project
  </Accordion>
  
  <Accordion title="Permission Issues">
    **Camera/Microphone not working:**
    1. Verify permissions are added to platform configuration files
    2. Check that permission descriptions are user-friendly
    3. Test on physical devices, not simulators
    4. Ensure app requests permissions at appropriate times
  </Accordion>
</AccordionGroup>
