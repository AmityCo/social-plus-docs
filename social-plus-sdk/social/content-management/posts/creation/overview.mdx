## Post Creation Workflow

<Steps>
  <Step title="Choose Content Type">
    Select the appropriate post type based on your content:
    - **Text**: Discussions, announcements, updates
    - **Image**: Photos, graphics, visual content
    - **Video**: Tutorials, entertainment, demonstrations
    - **File**: Documents, presentations, resources
    - **Poll**: Questions, surveys, community decisions
    - **Live Stream**: Real-time events, Q&As, broadcasts
  </Step>
  <Step title="Prepare Content">
    Optimize your content for the chosen format:
    - Compress images and videos
    - Write engaging captions
    - Add relevant hashtags or mentions
    - Include calls-to-action
  </Step>
  <Step title="Configure Settings">
    Set up post visibility and targeting:
    - Choose target (global, community, user)
    - Set visibility permissions
    - Configure moderation settings
    - Add metadata and tags
  </Step>
  <Step title="Publish & Monitor">
    Create the post and track engagement:
    - Publish to your audience
    - Monitor reactions and comments
    - Track performance metrics
    - Engage with responses
  </Step>
</Steps>

## Common Patterns

<AccordionGroup>
  <Accordion title="Multi-Media Posts">
    Create rich posts combining multiple content types:
    ```typescript
    // Create a parent post with child media posts
    const parentPost = await AmitySDK.createPost({
      type: 'text',
      text: 'Check out our latest product launch! ðŸš€',
      targetType: 'community',
      targetId: 'community-id'
    });
    
    // Add image child post
    const imageChild = await AmitySDK.createPost({
      type: 'image',
      parentId: parentPost.postId,
      images: [{ fileId: 'image-file-id' }]
    });
    
    // Add video child post
    const videoChild = await AmitySDK.createPost({
      type: 'video',
      parentId: parentPost.postId,
      videos: [{ fileId: 'video-file-id' }]
    });
    ```
  </Accordion>
  
  <Accordion title="Interactive Content">
    Create engaging posts that encourage interaction:
    ```typescript
    // Poll post with multiple options
    const pollPost = await AmitySDK.createPost({
      type: 'poll',
      text: 'What feature should we build next?',
      poll: {
        question: 'Vote for your favorite!',
        options: [
          { text: 'Dark mode' },
          { text: 'Video calling' },
          { text: 'File sharing' },
          { text: 'Advanced search' }
        ],
        multipleAnswers: false,
        closedIn: 7 * 24 * 60 * 60 * 1000 // 7 days
      }
    });
    ```
  </Accordion>
  
  <Accordion title="Community-Specific Posts">
    Target posts to specific communities:
    ```typescript
    // Create post in specific community
    const communityPost = await AmitySDK.createPost({
      type: 'text',
      text: 'Welcome to our community guidelines update!',
      targetType: 'community',
      targetId: 'community-id',
      metadata: {
        isAnnouncement: true,
        priority: 'high'
      }
    });
    ```
  </Accordion>
</AccordionGroup>

## Best Practices

<AccordionGroup>
  <Accordion title="Content Quality">
    - **Optimize Media**: Compress images and videos before upload
    - **Write Compelling Captions**: Engage your audience with interesting descriptions
    - **Use Relevant Hashtags**: Help users discover your content
    - **Include Call-to-Actions**: Encourage engagement and responses
  </Accordion>
  
  <Accordion title="Technical Considerations">
    - **File Size Limits**: Stay within platform limits for media uploads
    - **Error Handling**: Implement proper error handling for failed uploads
    - **Progress Tracking**: Show upload progress for large files
    - **Offline Support**: Cache content for offline creation
  </Accordion>
  
  <Accordion title="User Experience">
    - **Preview Mode**: Allow users to preview before publishing
    - **Draft Saving**: Save drafts automatically
    - **Edit Capabilities**: Enable post editing after creation
    - **Accessibility**: Ensure content is accessible to all users
  </Accordion>
</AccordionGroup>
