The **Notification Tray SDK Feature** enhances user engagement and connectivity within your application by providing comprehensive notification management capabilities. This feature allows users to receive notifications based on interactions from others, ensuring they stay informed and connected to their community.

Designed for seamless integration, the Notification Tray SDK ensures **real-time updates**, improved **user retention**, and a more engaging **social experience**.

## Data Models

### Notification Tray Item

<Tabs>
  <Tab title="Properties Overview">
    | Property | Type | Description |
    |----------|------|-------------|
    | `notificationId` | `String` | Unique identifier of notification item |
    | `lastSeenAt` | `Datetime` | Timestamp when the notification was last seen |
    | `lastOccurredAt` | `Datetime` | Timestamp when the notification last occurred |
    | `actors` | `List<AmityNotificationActors>` | Data of users that acted on this notification item |
    | `actorCount` | `Int` | Number of users that acted on this notification item |
    | `actionType` | `String` | Type of action that triggered this notification |
    | `trayItemCategory` | `String` | Category for when action is either "mention" or "reply" |
    | `targetId` | `String` | Object ID of the target |
    | `targetType` | `String` | Type of target for this action |
    | `referenceId` | `String` | Optional ObjectId of the reference |
    | `referenceType` | `String` | Type of reference for this action |
    | `parentId` | `String` | Optional ObjectId of parent |
    | `text` | `String` | Ready to render text without any templating |
    | `templatedText` | `String` | Ready to render text with templating for client interpretation |
    | `isSeen` | `Boolean` | Whether this notification item has been seen |
    | `isRecent` | `Boolean` | Whether this notification item is recent or older |
    | `users` | `List<AmityUser>` | List of users that acted on this notification |
  </Tab>
  
  <Tab title="Action Types">
    | Action Type | Description | Example Scenario |
    |-------------|-------------|------------------|
    | `post` | New post created | "John posted in Photography Community" |
    | `comment` | Comment on post | "Sarah commented on your post" |
    | `reply` | Reply to comment | "Mike replied to your comment" |
    | `reaction` | Post or comment reaction | "3 people liked your post" |
    | `mention` | User mentioned | "You were mentioned in a post" |
    | `follow` | User followed | "Anna started following you" |
    | `join` | Community joined | "5 new members joined your community" |
  </Tab>
  
  <Tab title="Target Types">
    | Target Type | Description | Use Case |
    |-------------|-------------|----------|
    | `post` | Post-related notification | Comments, reactions on posts |
    | `comment` | Comment-related notification | Replies to comments |
    | `community` | Community-related notification | New posts, member activities |
    | `user` | User-related notification | Follow notifications, mentions |
    | `channel` | Channel-related notification | Channel posts, discussions |
  </Tab>
</Tabs>

### Notification Tray Seen Status

<AccordionGroup>
  <Accordion title="Tray-Level Tracking" icon="eye">
    | Property | Type | Description |
    |----------|------|-------------|
    | `lastTraySeenAt` | `Datetime` | Timestamp when the tray was last seen |
    | `lastTrayOccurredAt` | `Datetime` | Timestamp when the last tray item occurred |
    | `isSeen` | `Boolean` | Whether the entire tray has been seen |
    
    **Usage**: Track when user last viewed the notification tray to show unread count badges
  </Accordion>
  
  <Accordion title="Item-Level Tracking" icon="check">
    | Property | Type | Description |
    |----------|------|-------------|
    | `notificationId` | `String` | Unique identifier of the notification |
    | `isSeen` | `Boolean` | Whether this specific notification has been seen |
    | `lastSeenAt` | `Datetime` | When this notification was marked as seen |
    
    **Usage**: Track individual notification read status for granular control
  </Accordion>
</AccordionGroup>

## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion title="Notifications Not Appearing" icon="triangle-exclamation">
    **Possible Causes**:
    - Network connectivity issues
    - Notification repository not properly initialized
    - User permissions not granted
    
    **Solutions**:
    1. Check network connectivity and API endpoints
    2. Verify notification repository initialization
    3. Ensure user has proper permissions
    4. Test with manual notification refresh
    5. Check console for error messages
  </Accordion>
  
  <Accordion title="Seen Status Not Updating" icon="eye-slash">
    **Possible Causes**:
    - API call failures
    - Cross-device synchronization issues
    - Incorrect notification item ID handling
    - Race conditions in state updates
    
    **Solutions**:
    1. Verify markAsSeen API calls are succeeding
    2. Check for proper error handling
    3. Ensure notification item IDs are correct
    4. Implement retry logic for failed requests
    5. Test cross-device synchronization
  </Accordion>
  
  <Accordion title="Performance Issues" icon="gauge">
    **Possible Causes**:
    - Loading too many notifications at once
    - Inefficient UI updates
    - Memory leaks from unsubscribed observers
    
    **Solutions**:
    1. Implement proper pagination
    2. Use efficient list rendering techniques
    3. Dispose of observers properly
    4. Monitor memory usage
  </Accordion>
</AccordionGroup>
