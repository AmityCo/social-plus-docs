## Feature Overview

The Unread Status system provides comprehensive tracking of message read and delivery states, offering real-time insights into user engagement and message interaction patterns. This feature set enables sophisticated chat interfaces with accurate unread indicators, delivery confirmations, and user activity tracking.

  <Tab title="Basic Setup">
    **Get started with unread status tracking**
    
    Begin with essential unread count display and basic read status management for your chat interface.

    ### Step 1: Channel Unread Display
    ```typescript
    // Display unread counts in channel list
    const ChannelList = () => {
      return channels.map(channel => (
        <ChannelItem
    ```

    ### Step 2: Basic Read Tracking
    ```typescript
    // Mark messages as read when viewed
    const MessageList = () => {
      useEffect(() => {
        // Mark latest message as read when component mounts
        if (messages.length > 0) {
          messages[0].markRead();
        }
      }, [messages]);
    };
    ```

    ### Step 3: Total Unread Badge
    ```typescript
    // Show total unread count in app navigation
    const AppNavigation = () => {
      const [totalUnread, setTotalUnread] = useState(0);
      
      useEffect(() => {
        const unsubscribe = ChannelRepository.getTotalChannelsUnread(
          ({ data }) => setTotalUnread(data?.unreadCount || 0)
        );
        return unsubscribe;
      }, []);
      
      return <TabBadge count={totalUnread} />;
    };
    ```

  <Tab title="Advanced Features">
    **Implement comprehensive status tracking**
    
    Add sophisticated delivery tracking, user analytics, and real-time synchronization for professional chat experiences.

    ### Real-time Receipt Sync
    ```typescript
    // Start receipt sync when entering channel
    useEffect(() => {
      SubChannelRepository.startMessageReceiptSync(subChannelId);
      
      return () => {
        SubChannelRepository.stopMessageReceiptSync(subChannelId);
      };
    }, [subChannelId]);
    ```

    ### Delivery Status Tracking
    ```typescript
    // Track message delivery and read users
    const MessageStatus = ({ message }) => {
      const [readUsers, setReadUsers] = useState([]);
      const [deliveredUsers, setDeliveredUsers] = useState([]);
      
      useEffect(() => {
        // Get users who read the message
        MessageRepository.getReadUsers({ messageId: message.messageId })
          .then(({ data }) => setReadUsers(data));
          
        // Get users who received the message
        MessageRepository.getDeliveredUsers({ messageId: message.messageId })
          .then(({ data }) => setDeliveredUsers(data));
      }, [message.messageId]);
      
      return <StatusIndicator read={readUsers.length} delivered={deliveredUsers.length} />;
    };
    ```

    ### Smart Read Detection
    ```typescript
    // Automatically mark messages as read when they appear in viewport
    const useVisibilityTracking = (message) => {
      const [isVisible, setIsVisible] = useState(false);
      
      useEffect(() => {
        if (isVisible && !message.isRead) {
          message.markRead();
        }
      }, [isVisible, message]);
      
      return { isVisible, setIsVisible };
    };
    ```
  </Tab>

    ```typescript
    // Mark multiple channels as read
    const markAllChannelsRead = async (channelIds) => {
      const promises = channelIds.map(async (channelId) => {
        const messages = await getLatestMessages(channelId);
        return messages[0]?.markRead();
      });
      
      await Promise.all(promises);
    };
    ```

    ### Custom Status Filters
    ```typescript
    // Advanced user filtering for read/delivery status
    const getMessageStatusWithFilters = async (messageId) => {
      const readUsers = await MessageRepository.getReadUsers({
        messageId,
        memberships: ['member', 'muted'], // Exclude banned users
        limit: 100
      });
      
      return {
        readCount: readUsers.data.length,
        readUsers: readUsers.data
    ```

## Best Practices

<AccordionGroup>
  <Accordion title="Performance Optimization" icon="gauge-high">
    **Optimize status tracking for scale**
    
    - Cache unread counts to minimize API calls during UI rendering
    - Use efficient data structures for tracking multiple channel states
    - Implement lazy loading for read/delivery user lists
    - Batch status updates to reduce network overhead
    
    Performance optimization ensures smooth operation even with high message volume and many active channels.
  </Accordion>

  <Accordion title="User Experience Design" icon="user-check">
    **Create intuitive status interfaces**
    
    - Design clear visual hierarchy for different types of unread status
    - Provide immediate feedback when users interact with status features
    - Use consistent iconography and colors across all status indicators
    - Implement smooth animations for status changes to avoid jarring updates
    
    Thoughtful UX design makes status information helpful rather than overwhelming.
  </Accordion>

  <Accordion title="Resource Management" icon="memory">
    **Manage tracking resources efficiently**
    
    - Start receipt sync only when users are actively viewing channels
    - Stop receipt sync when users leave channels to conserve resources
    - Implement proper cleanup when components unmount
    - Monitor memory usage when tracking many channels simultaneously
    
    Efficient resource management prevents performance degradation and battery drain.
  </Accordion>

  <Accordion title="Error Handling" icon="triangle-exclamation">
    **Handle status tracking failures gracefully**
    
    - Provide fallback indicators when status data is unavailable
    - Implement retry logic for failed status requests
    - Handle network disconnections without breaking status tracking
    - Show appropriate loading states during status data retrieval
    
    Robust error handling maintains user trust even when data is temporarily unavailable.
  </Accordion>
</AccordionGroup>
