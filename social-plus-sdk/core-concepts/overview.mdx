# Core Concepts

Understanding these core concepts is essential for building successful applications with Social+. These concepts apply across all platforms and SDK implementations.

## Architecture Overview

Social+ SDK uses a client-server architecture with real-time synchronization:

<Card title="Key Components" icon="building">
- **Client SDK**: Your application interface to Social+ services
- **Session Management**: Authentication and connection handling  
- **Live Objects**: Real-time data synchronization
- **Repository Pattern**: Organized data access layer
</Card>

## Quick Start Guide

<Steps>
<Step title="Initialize SDK">
Set up your client with proper authentication:

```typescript
const client = Client.create({
  apiKey: 'your-api-key',
  region: 'sg'
});
```
</Step>

<Step title="Handle Session State">
Monitor authentication status:

```typescript
client.onSessionStateChange((state) => {
  switch (state) {
    case 'established':
      // User is authenticated, SDK is ready
      break;
    case 'tokenExpired':
      // Refresh authentication token
      break;
  }
});
```
</Step>

<Step title="Implement Error Handling">
Set up comprehensive error management:

```typescript
try {
  const result = await client.createPost(postData);
} catch (error) {
  if (error.code === 'PERMISSION_DENIED') {
    // Handle permission error
  }
}
```
</Step>

<Step title="Use Live Objects">
Subscribe to real-time data updates:

```typescript
const livePost = client.getPost(postId);
livePost.subscribe((post) => {
  // Handle real-time post updates
});
```
</Step>
</Steps>

## Best Practices

### Session Management
- Always implement session state monitoring
- Handle token expiration gracefully  
- Use secure logout when appropriate

### Error Handling
- Implement specific error handling for different error types
- Provide user-friendly error messages
- Log errors for debugging and monitoring

### Performance
- Use Live Objects for real-time features
- Implement proper pagination for large data sets
- Cache data appropriately to reduce network requests

### Security
- Never expose API keys in client code
- Always use authentication tokens in production
- Implement proper permission checks
