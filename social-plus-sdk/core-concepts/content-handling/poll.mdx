## Create a Poll

Create engaging polls with customizable settings and up to 10 answer options.

<Tip>
  Design polls with clear, concise questions and meaningful answer options to
  maximize user engagement and response quality.
</Tip>

### Configuration Options

<AccordionGroup>
  <Accordion title="Poll Structure" icon="file-text">
    | Parameter | Type | Description | Limits |
    |-----------|------|-------------|---------| | `question` | String | Poll
    question text | Max 500 characters | | `answers` | Array | Available answer
    options | 2-10 options, max 200 chars each |
  </Accordion>

  <Accordion title="Poll Behavior" icon="cog">
    | Parameter | Type | Description | Limits |
    |-----------|------|-------------|---------| | `answerType` | Enum | Single
    or multiple choice | `Single` (default), `Multiple` | | `timeToClosePoll` |
    Number | Poll duration in seconds | Default: 30 days |
  </Accordion>
</AccordionGroup>

## Delete Polls

Permanently remove polls from the system. This action cannot be undone.

<Warning>
  **Destructive Action**: Poll deletion is permanent and removes all associated
  votes and data.
</Warning>

<CodeGroup>
```swift iOS 
do {
   let result = try await pollRepository.deletePoll(withId: "<poll-id>")
} catch {
    // Handle error here
}
```

```kotlin Android
fun deletePoll(
    pollRepository: AmityPollRepository,
    poll: AmityPoll
) {
    pollRepository.deletePoll(pollId = poll.getPollId())
        .doOnComplete {
            // Void
        }
        .doOnError {
            // Exception
        }
        .subscribe()
}
```

```typescript TypeScript
import { PollRepository } from "@amityco/ts-sdk";

async function deletePoll() {
  const isDeleted = await PollRepository.deletePoll("pollId");

  return isDeleted;
}
```

```dart Flutter
void deletePoll() {
  AmitySocialClient.newPollRepository()
      .deletePoll(pollId: 'pollId')
      .then((value) {
    //success
  }).onError((error, stackTrace) {
    //handle error
  });
}
```

</CodeGroup>

## Best Practices

<AccordionGroup>
  <Accordion title="Poll Design" icon="palette">
    - Keep questions clear and concise (under 500 characters) - Provide 2-6
    answer options for better user experience - Use descriptive answer text
    (avoid generic "Option 1", "Option 2") - Consider your audience when setting
    poll duration
  </Accordion>

{" "}

<Accordion title="User Experience" icon="user">
  - Indicate poll expiration time clearly - Provide visual feedback when votes
  are submitted - Handle edge cases (poll closed, already voted) - Show
  real-time vote counts when appropriate
</Accordion>

{" "}

<Accordion title="Error Handling" icon="exclamation-triangle">
  - Check poll status before allowing votes - Validate user permissions for
  management actions - Provide clear error messages for failed operations -
  Implement retry logic for network failures
</Accordion>

  <Accordion title="Performance" icon="gauge">
    - Cache poll results to reduce API calls - Implement efficient polling for
    live updates - Consider pagination for polls with many votes - Optimize for
    mobile network conditions
  </Accordion>
</AccordionGroup>
