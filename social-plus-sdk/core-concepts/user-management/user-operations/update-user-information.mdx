---
title: "Update User Information"
description: "Update user profiles, avatars, and metadata using the social.plus SDK"
---

Update the authenticated user's profile, including display name, description, avatar, and custom metadata. Only the current user can update their own profile through the client SDK.

Before setting an avatar, upload the image file first. Refer to the [Image Upload guide](/social-plus-sdk/core-concepts/content-handling/files-images-and-videos/image-handling#image-upload) for detailed instructions.

<Info>
**User Authorization**: Only the currently authenticated user can update their own profile information. Administrators may have additional privileges through the Console.
</Info>

## Parameters

| Field | Required | Description |
| --- | --- | --- |
| Display name | No | User-facing profile name. |
| Description | No | Short profile description or bio. |
| Avatar | No | Uploaded image reference used as the user's avatar. |
| Metadata | No | Custom key-value data associated with the user profile. |

## Code examples

Call the platform update method with only the profile fields you want to change.

<CodeGroup>
```swift iOS
// Build the current user data
let options = AmityUserUpdateOptions()
options.setDisplayName("Batman")
options.setUserDescription("Hero that Gotham needs")
options.setAvatar(nil)

// Add custom metadata
let metadata: [String: Any] = ["role": "superhero", "city": "Gotham"]
options.setUserMetadata(metadata)

// Update the current user from the options
do {
    try await client.editUser(options)
    print("User updated successfully")
} catch {
    print("Update failed: \(error.localizedDescription)")
}
```

```kotlin Android
fun updateUser(uploadedAvatar: AmityImage) {
    // Update basic profile information
    AmityCoreClient.editUser()
        .displayName(displayName = "Batman")
        .description(description = "Hero that Gotham needs")
        .avatar(avatar = uploadedAvatar)
        .metadata(metadata = JsonObject().apply {
            addProperty("role", "superhero")
            addProperty("city", "Gotham")
        })
        .build()
        .apply()
        .doOnSuccess { user: AmityUser ->
            // Handle successful update
            Log.d("UserUpdate", "User updated: ${user.getDisplayName()}")
        }
        .doOnError { error ->
            // Handle error
            Log.e("UserUpdate", "Update failed: ${error.message}")
        }
        .subscribe()
}
```

```typescript TypeScript
import { UserRepository } from '@amityco/ts-sdk';

// Update user profile information
async function updateUser() {
    try {
        const { data: updatedUser } = await UserRepository.updateUser('userId', {
            displayName: 'Batman',
            description: 'Hero that Gotham needs',
        });
        
        console.log('User updated successfully:', updatedUser.displayName);
        return updatedUser;
    } catch (error) {
        console.error('Update failed:', error);
        throw error;
    }
}
```

## Best practices

### Performance optimization

<Tip>
When updating multiple user properties, combine them into a single update operation instead of making separate calls.
</Tip>

<Warning>
Be mindful of update frequency to avoid hitting rate limits. Consider implementing debouncing for real-time input fields.
</Warning>
