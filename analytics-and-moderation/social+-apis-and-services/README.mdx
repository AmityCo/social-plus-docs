# Social+ APIs and Services

The Social+ APIs provide comprehensive administrative capabilities for server-to-server integration, enabling advanced moderation, analytics, and platform management beyond the console interface.

<Info>
Many administrative functions are only accessible through APIs, making them essential for custom integrations and automated workflows.
</Info>

## Authentication Methods

<Tabs>
  <Tab title="Admin Access">
    ### Admin API Access Token

    For server-to-server API calls with full administrative privileges:

    <Steps>
      <Step title="Generate Token">
        Navigate to Console → **Settings** → **Security** → **Admin API Access Token**
      </Step>
      <Step title="Configure Permissions">
        Set appropriate scopes and expiration for your integration
      </Step>
      <Step title="Secure Storage">
        Store the token securely and rotate regularly
      </Step>
    </Steps>

    <Warning>
    Admin tokens provide full access to your application. Never expose them in client-side code or public repositories.
    </Warning>

    ```bash
    curl -X GET "https://api.amity.co/api/v3/communities" \
      -H "Authorization: Bearer YOUR_ADMIN_TOKEN"
    ```
  </Tab>

  <Tab title="User Access">
    ### User Access Token

    For user-context API calls with scoped permissions:

    <Steps>
      <Step title="Register Session">
        Call the [Session API](https://api-docs.amity.co/#/Session/post_api_v3_sessions) to create a user session
      </Step>
      <Step title="Retrieve Token">
        Extract the `accessToken` from the session response
      </Step>
      <Step title="Make API Calls">
        Use the token for user-scoped operations
      </Step>
    </Steps>

    ```javascript
    // Register user session
    const response = await fetch('https://api.amity.co/api/v3/sessions', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'x-api-key': 'YOUR_API_KEY'
      },
      body: JSON.stringify({
        userId: 'user123',
        deviceId: 'device456'
      })
    });

    const { accessToken } = await response.json();
    ```

    <Info>
    User Access tokens are valid for 30 days but will be invalidated if a different user ID registers with the same device.
    </Info>
  </Tab>
</Tabs>

## Token Management

<AccordionGroup>
  <Accordion title="Token Security Best Practices">
    - **Secure Storage**: Use environment variables or secure key management
    - **Regular Rotation**: Implement automated token rotation
    - **Scope Limitation**: Use minimum required permissions
    - **Monitoring**: Log and monitor API usage patterns
    - **Revocation**: Implement emergency token revocation procedures
  </Accordion>

  <Accordion title="Revoking User Access Tokens">
    Admin tokens can revoke user access tokens using the [Revoke API](https://api-docs.amity.co/#/Session/delete_api_v4_sessions):

    ```bash
    curl -X DELETE "https://api.amity.co/api/v4/sessions" \
      -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
      -H "Content-Type: application/json" \
      -d '{"userId": "user123"}'
    ```
  </Accordion>

  <Accordion title="Token Validation">
    Validate tokens before making API calls:

    ```javascript
    const validateToken = async (token) => {
      try {
        const response = await fetch('https://api.amity.co/api/v3/users/me', {
          headers: { 'Authorization': `Bearer ${token}` }
        });
        return response.ok;
      } catch (error) {
        return false;
      }
    };
    ```
  </Accordion>
</AccordionGroup>
