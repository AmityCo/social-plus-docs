## Overview

The new Live Stream Management page displays room-based livestreams created after migration to the room-based architecture. This system supports two stream types — **Direct streams** (single broadcaster) and **Co-streams** (multiple participants) — and introduces features like product tagging and enhanced moderation visibility.

<img
  src="/images/analytics-and-moderation/console-live-stream-management.png"
  alt="Live Stream Management Dashboard"
/>

<Warning>
**Migration Notice**: Live stream (New) shows room-based livestreams created after migration. For accounts still using Legacy stream, livestreams created before migration will remain available under [Live stream (Legacy)](/analytics-and-moderation/console/management/live-stream-management).
</Warning>

The live stream list displays all streams with the following information at a glance:

| Column | Description |
|--------|-------------|
| **Thumbnail** | Preview image of the live stream |
| **Live stream title** | The name of the stream |
| **Live stream type** | Stream type — `Direct stream` or `Co-stream` |
| **Creator** | The user who created the stream |
| **Products tagged** | Number of products tagged to the stream |
| **Broadcast status** | Current status — `Not started`, `Live`, `Finished`, or `Terminated` |
| **Duration** | Total broadcast duration (displayed after stream ends) |
| **Moderation** | AI content moderation result (e.g., `Passed`, `Violence`) |
| **Created at** | Timestamp of stream creation |

You can filter streams by **broadcast status** using the dropdown filter at the top of the page.

## Quick Start

<Steps>
  <Step title="Navigate to Live Streams">
    Go to **Console → Live stream → Live stream** from the left sidebar navigation.
  </Step>

  <Step title="Create a New Stream">
    Click the **Create live stream** button in the top-right corner.
  </Step>

  <Step title="Enter Stream Details">
    Fill in the title, description (optional), and upload a thumbnail image (optional).
  </Step>

  <Step title="Publish">
    Review all details and click **Create live stream** to generate the stream.
  </Step>
</Steps>

## Creating a Live Stream

To create a new live stream, click the **Create live stream** button from the management dashboard. Streams created from the Console are configured as **direct streams**, which support basic broadcast features. Advanced live features (such as co-streaming) are available when starting a stream from the app.

<img
  src="/images/analytics-and-moderation/console-live-stream-create.png"
  alt="Create Live Stream Form"
/>

### Stream Details

<AccordionGroup>
  <Accordion title="Title" icon="heading">
    **Type:** Text
    **Required:** Yes
    **Character limit:** 150

    A clear, descriptive name for the live stream event. This is displayed in the stream list and detail pages.
  </Accordion>

  <Accordion title="Description" icon="align-left">
    **Type:** Text
    **Required:** No
    **Character limit:** 500

    Additional context about the stream event. Helps viewers understand what to expect.
  </Accordion>

  <Accordion title="Thumbnail" icon="image">
    **Type:** Image upload
    **Required:** No
    **Recommended size:** 1280×720
    **Max file size:** 2MB

    A preview image for the stream. Displayed in the stream list and as the cover before broadcast begins.
  </Accordion>
</AccordionGroup>

<Note>
**Console vs App**: Streams created from the Console are always configured as **direct streams**. To create co-streams or access advanced live features, start the stream from your application using the SDK or API.
</Note>

## Viewing Stream Details

Click on any stream row in the list to view its full details. The detail page shows:

<img
  src="/images/analytics-and-moderation/console-live-stream-detail.png"
  alt="Live Stream Detail View"
/>

<Tabs>
  <Tab title="Stream Information">
    - **Title**: Stream name with broadcast status badge
    - **Description**: Event description (or "No description added")
    - **Thumbnail**: Preview image
    - **Live stream type**: Direct stream or Co-stream
    - **Stream ID**: Unique identifier (with copy button)
  </Tab>

  <Tab title="Feed Details">
    - **Creator**: Username of the stream creator
    - **Creator ID**: Unique user identifier
    - **Created at**: Timestamp of stream creation
  </Tab>

  <Tab title="Broadcast Info">
    - **Started at**: When the broadcast began
    - **Ended at**: When the broadcast ended
    - **Duration**: Total broadcast length
    - **Video player**: Playback of the recorded stream
  </Tab>

  <Tab title="Moderation">
    - **Moderation status**: AI content moderation result
    - **Flags**: Specific content flags (e.g., Violence, Nudity)
    - **Severity indicators**: Color-coded severity levels
  </Tab>
</Tabs>

### Stream Actions

From the detail page, you can:

- **Edit stream**: Update title, description, and thumbnail using the edit button
- **Delete stream**: Permanently remove the stream using the **Delete stream** button
- **Copy stream ID**: Copy the unique stream identifier to clipboard

<Warning>
**Deletion is permanent**: Deleting a stream cannot be undone. All associated data, including recordings and moderation history, will be removed.
</Warning>

## Broadcast Status

The broadcast status reflects the current state of the stream:

<AccordionGroup>
  <Accordion title="Not Started" icon="clock">
    The stream has been created but broadcasting has not begun. The stream is waiting for the broadcaster to go live.
  </Accordion>

  <Accordion title="Live" icon="signal">
    The stream is currently broadcasting. Viewers can join and watch the live content in real-time.
  </Accordion>

  <Accordion title="Finished" icon="check">
    The broadcast has ended normally. The recorded stream is available for playback and the final duration is displayed.
  </Accordion>

  <Accordion title="Terminated" icon="ban">
    The stream was forcibly ended, typically due to a moderation action or policy violation. This status indicates the stream was stopped before the broadcaster chose to end it.
  </Accordion>
</AccordionGroup>

## Content Moderation

Live streams are automatically reviewed by AI content moderation. Results are displayed both in the list view and on the stream detail page.

<Tabs>
  <Tab title="Moderation Results">
    | Status | Indicator | Description |
    |--------|-----------|-------------|
    | **Passed** | 🟢 Green | Stream content passed all moderation checks |
    | **Violence** | 🟡 Yellow / 🔴 Red | Violent content detected — severity varies |
    | **Nudity** | 🔴 Red | Adult or explicit content detected |
    | **Other flags** | Various | Additional content policy violations |
  </Tab>

  <Tab title="Moderation Actions">
    - Streams flagged with severe violations may be **automatically terminated**
    - Review flagged streams promptly from the management list
    - Sort by moderation status to prioritize review of flagged content
    - Access detailed moderation information from the stream detail page
  </Tab>
</Tabs>

<Info>
**Moderation Priority**: Focus on red-flagged streams first, as they indicate critical content that may violate community guidelines or require immediate action.
</Info>

## Product Tagging

Live streams support product tagging for shoppable live commerce experiences. The **Products tagged** column in the list view shows how many products are associated with each stream.

<AccordionGroup>
  <Accordion title="How Product Tagging Works" icon="tag">
    - Products are tagged to streams via the SDK or API
    - Tagged products appear in the stream detail page
    - Viewers can browse and purchase tagged products during or after the stream
    - The Console displays the count of tagged products in the list view
  </Accordion>

  <Accordion title="Use Cases" icon="shopping-cart">
    - **Live Shopping Events**: Tag featured products for instant purchase during broadcasts
    - **Product Demos**: Associate demonstrated products for viewer convenience
    - **Influencer Streams**: Enable product recommendations with direct links
  </Accordion>
</AccordionGroup>

## Best Practices

<AccordionGroup>
  <Accordion title="Stream Setup" icon="gear">
    - **Use descriptive titles**: Help viewers find and understand your stream content
    - **Upload thumbnails**: Streams with thumbnails (recommended: 1280×720) get better engagement
    - **Add descriptions**: Provide context about what viewers can expect
    - **Plan ahead**: Create streams in advance and share details before going live
  </Accordion>

  <Accordion title="Monitoring & Moderation" icon="shield">
    - **Monitor broadcast status**: Keep the Console open during live events to track stream health
    - **Review moderation flags**: Check flagged streams promptly and take appropriate action
    - **Use status filters**: Filter by broadcast status to quickly find active or problematic streams
    - **Track terminated streams**: Investigate terminated streams to understand policy violations
  </Accordion>

  <Accordion title="Product Tagging Strategy" icon="tag">
    - **Tag before going live**: Ensure products are tagged before the broadcast starts
    - **Limit product count**: Focus on featured items rather than overwhelming viewers
    - **Update tags post-stream**: Keep product associations current for replay viewers
  </Accordion>

  <Accordion title="Performance & Reliability" icon="gauge">
    - **Bandwidth planning**: Ensure sufficient upload capacity for high-quality streams
    - **Test before events**: Verify stream configuration with a test broadcast
    - **Monitor duration**: Track stream lengths for capacity and billing planning
  </Accordion>
</AccordionGroup>

## Troubleshooting

| Issue | Likely Cause | Resolution |
|-------|--------------|------------|
| Cannot create stream | Missing required title field | Enter a stream title (required, max 150 characters) |
| Thumbnail upload fails | Image exceeds 2MB or wrong format | Use an image under 2MB in a supported format (PNG, JPG) |
| Stream shows "Not started" | Broadcaster hasn't gone live yet | Wait for the broadcaster to start the stream from the app |
| Stream terminated unexpectedly | Content moderation flag triggered | Review moderation details on the stream detail page |
| Products not showing | Products tagged via SDK/API not yet synced | Refresh the page; verify product tagging via API |
| Co-stream not in Console | Co-streams are created from the app | Co-streams created from the app will appear in the list automatically |
| Duration not displaying | Stream hasn't finished yet | Duration appears after the broadcast ends |
| Cannot find legacy streams | Legacy streams are in a separate view | Go to **Live stream → Live stream (legacy)** in the sidebar |
