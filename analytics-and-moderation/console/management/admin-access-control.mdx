## Roles Overview

| Role | Scope | Key Capabilities | Safeguards |
|------|-------|------------------|------------|
| Super Admin | Full platform governance | Create / edit / delete admins, manage Secure Mode & tokens | Multi-person ownership, audit logs |
| General Admin | Operational tasks (no high-risk auth) | Configure content, moderate, create admins (if allowed), cannot manage Secure Mode tokens | Scoped permissions disabled by default |

<Warning>Grant Super Admin only to a minimal, trusted set. Rotate reviews quarterly.</Warning>

## Permission Definitions

<AccordionGroup>
  <Accordion title="Access Secure Mode">Allows generation & use of elevated Admin Tokens for secure API authentication. Restrict to security / platform engineering.</Accordion>
  <Accordion title="Manage Admin">Create (and where applicable edit) other admin accounts. All new accounts start with permissions disabled (principle of deny-by-default).</Accordion>
  <Accordion title="Principle of Least Privilege">Grant only the minimal permissions required for current responsibilities; schedule periodic entitlement reviews.</Accordion>
  <Accordion title="Separation of Duties">Separate token generation (Secure Mode) from daily moderation operations to reduce abuse risk.</Accordion>
</AccordionGroup>

## Core Workflows

<Tabs>
  <Tab title="View Admins">Open Admin Users page → scan role distribution & recent changes.</Tab>
  <Tab title="Edit Permissions">Select admin → adjust toggles → save → log rationale (date, approver).</Tab>
  <Tab title="Create Admin">Click Create → enter identity info → assign minimal permissions → communicate onboarding steps.</Tab>
  <Tab title="Restrict Access">Remove unneeded Secure Mode or Manage Admin from dormant / role-changed users.</Tab>
  <Tab title="Audit Cycle">Monthly: export admin list → compare to HR roster → revoke stale accounts.</Tab>
</Tabs>

## Edit Existing Admin Permissions

<Steps>
  <Step title="Locate">Navigate to Admin Users page.</Step>
  <Step title="Select">Click "Edit name & Permissions" beside target admin.</Step>
  <Step title="Adjust">Toggle required permissions; update display name / email if needed.</Step>
  <Step title="Review Impact">Confirm no conflicting segregation-of-duties (e.g., adding Secure Mode plus broad moderation without approval).</Step>
  <Step title="Save & Log">Click Save Changes; record rationale in internal change log.</Step>
</Steps>

## Create a New Admin

<Steps>
  <Step title="Open Creation">On Admin Users page click "Create new admin".</Step>
  <Step title="Enter Identity">Provide username, email, and basic profile info.</Step>
  <Step title="Assign Minimal Permissions">Enable only what's needed for day 1; leave Secure Mode off unless justified.</Step>
  <Step title="Confirm & Create">Click Create Admin.</Step>
  <Step title="Onboard">Share guidelines & acceptable use; schedule 30‑day permission review.</Step>
</Steps>

<Info>All permissions are disabled by default for new admins—activate intentionally.</Info>

## Governance & Risk Controls

<AccordionGroup>
  <Accordion title="Quarterly Review">Export list → verify employment status & role alignment → revoke excess rights.</Accordion>
  <Accordion title="Joiner / Mover / Leaver Process">Tie admin creation & removal to HR events; revoke within SLA (e.g., 24h of departure).</Accordion>
  <Accordion title="Dual Control">Require 2 Super Admin approvals for Secure Mode enablement or new Super Admin assignment.</Accordion>
  <Accordion title="Change Logging">Maintain external log: date, admin modified, fields changed, approver.</Accordion>
  <Accordion title="Anomaly Detection">Flag sudden spike in General Admin count or multiple permission escalations in short window.</Accordion>
</AccordionGroup>

## Metrics & Monitoring

| Metric | Description | Healthy Signal | Action Trigger |
|--------|-------------|----------------|----------------|
| Super:General Ratio | Super Admins / Total Admins | < 0.15 | ≥ 0.25 → review downsizing |
| Dormant Admins | No login in 45 days | Near 0 | Any > 0 → assess removal |
| Permission Change Frequency | Changes per month | Stable trend | Spike → investigate cause |
| Secure Mode Token Holders | Count with token ability | Minimal & justified | Growth without tickets |
| Orphan Accounts | Admins lacking active employee record | 0 | >0 → immediate revoke |

## Troubleshooting

| Issue | Likely Cause | Resolution |
|-------|--------------|------------|
| Cannot edit another admin | Not a Super Admin | Escalate to Super Admin or adjust role mapping |
| Unauthorized token creation | Over-granted Secure Mode | Revoke permission; rotate tokens; audit logs |
| Excess Super Admins | Legacy grandfathered users | Conduct entitlement review & downgrade | 
| Missing audit trail | Informal changes not logged | Institute mandatory change log template |
| High dormant accounts | Offboarding gaps | Integrate with HRIS; automate deactivation |
