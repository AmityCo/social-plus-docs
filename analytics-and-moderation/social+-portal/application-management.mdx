# Application Management

The Social+ Portal provides comprehensive application management capabilities, allowing you to create, configure, and manage up to 50 Social+ applications per organization with regional deployment options and advanced organizational controls.

<Info>
Application Management is accessible through the Social+ Portal at [portal.amity.co](https://portal.amity.co). Each organization can manage up to 50 applications with dedicated regional configurations.
</Info>

## Creating Applications

<Tabs>
  <Tab title="New Application">
    ### Application Creation Process

    <Steps>

      </Step>
      <Step title="Configure Basic Settings">

    ### Application Configuration

    <AccordionGroup>
      <Accordion title="Basic Information">
        **Required Fields:**
        - **Application Name**: Unique identifier for your application
        - **Description**: Brief description of the application purpose
        - **Environment**: Development, Staging, or Production
        - **Application Type**: Social, Chat, or Combined

        ```json
        {
          "name": "My Social Platform",
          "description": "Community platform for our organization",
          "environment": "production",
          "type": "combined",
          "region": "us-east-1"
        }
        ```

## Application Settings

<Tabs>
  <Tab title="API Configuration">
    ### API Key Management

    <AccordionGroup>
      <Accordion title="API Key Generation">
        **Creating API Keys:**
        - Generate secure API keys for each environment
        - Set appropriate permissions and scopes
        - Configure expiration dates and rotation
        - Implement key usage monitoring

        ```bash
        # Example API key configuration
        {
          "keyId": "sk_live_abc123...",
          "environment": "production",
          "permissions": ["read", "write", "admin"],
          "expiresAt": "2024-12-31T23:59:59Z",
          "usage": {
            "dailyLimit": 100000,
            "currentUsage": 45000
          }
        }
        ```
      </Accordion>

      <Accordion title="Security Best Practices">
        **API Key Security:**
        - Store keys securely in environment variables
        - Use different keys for different environments
        - Implement key rotation policies
        - Monitor key usage and detect anomalies
        - Restrict key permissions to minimum required

        **Access Control:**
        - IP whitelisting for production keys
        - Rate limiting per key
        - Audit logging for key usage
        - Automatic key revocation on suspicious activity
      </Accordion>

      <Accordion title="Integration Examples">
        **SDK Integration:**
        ```javascript
        // JavaScript SDK initialization
        import { AmitySDK } from 'amity-sdk';

        const client = AmitySDK.create({
          apiKey: process.env.AMITY_API_KEY,
          region: 'us-east-1',
          environment: 'production'
        });
        ```

        **REST API Usage:**
        ```bash
        curl -X GET "https://api.amity.co/api/v3/users" \
          -H "Authorization: Bearer YOUR_API_KEY" \
          -H "Content-Type: application/json"
        ```
      </Accordion>
    </AccordionGroup>
  </Tab>

    ### Webhook Configuration

    <AccordionGroup>
      <Accordion title="Webhook Setup">
        **Configuration Steps:**
        - Define webhook endpoint URLs
        - Select event types to monitor
        - Configure authentication methods
        - Set up retry policies and error handling

        **Event Types:**
        - User registration and profile updates
        - Content creation and moderation
        - Community activity and engagement
        - System events and errors
      </Accordion>

      <Accordion title="Webhook Security">
        **Security Measures:**
        - HMAC signature verification
        - IP whitelisting for webhook endpoints
        - HTTPS-only webhook URLs
        - Request timeout configuration
        - Retry logic with exponential backoff

        ```javascript
        // Webhook signature verification
        const crypto = require('crypto');

        function verifyWebhookSignature(payload, signature, secret) {
          const expectedSignature = crypto
            .createHmac('sha256', secret)
            .update(payload, 'utf8')
            .digest('hex');
          
          return crypto.timingSafeEqual(
            Buffer.from(signature),
            Buffer.from(expectedSignature)
          );
        }
        ```
      </Accordion>

      <Accordion title="Webhook Testing">
        **Testing and Validation:**
        - Use webhook testing tools
        - Implement payload validation
        - Test error handling scenarios
        - Monitor webhook delivery success rates
        - Set up alerting for failed deliveries
      </Accordion>

## Best Practices

<AccordionGroup>
  <Accordion title="Application Naming">
    **Naming Conventions:**
    - Use descriptive, meaningful names
    - Include environment identifiers (dev, staging, prod)
    - Follow organizational naming standards
    - Avoid special characters and spaces
    - Keep names concise but informative

    **Examples:**
    - `myapp-production-us`
    - `internal-chat-staging`
    - `community-platform-eu`
  </Accordion>

  <Accordion title="Environment Management">
    **Environment Strategy:**
    - Separate applications for dev/staging/production
    - Use consistent configuration across environments
    - Implement proper promotion workflows
    - Maintain environment-specific API keys
    - Document environment differences

    **Configuration Management:**
    - Use infrastructure-as-code tools
    - Maintain configuration version control
    - Implement automated deployment pipelines
    - Test configurations in staging environments
  </Accordion>

  <Accordion title="Security Management">
    **Security Best Practices:**
    - Regular API key rotation
    - Implement least-privilege access
    - Monitor and audit all activities
    - Use secure communication channels
    - Keep security patches up to date

    **Compliance Considerations:**
    - Understand data residency requirements
    - Implement proper data retention policies
