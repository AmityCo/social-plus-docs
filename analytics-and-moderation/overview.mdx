## Key Capabilities

### **Community Moderation**

- **Content Moderation**: Review, approve, and manage user-generated content
- **User Management**: Handle user behavior, roles, and access controls
- **AI-Powered Tools**: Automated content filtering and detection
- **Reporting Systems**: Comprehensive flagging and review workflows

### **Analytics & Insights**

- **User Activity**: Track engagement, participation, and growth metrics
- **Content Analytics**: Monitor post performance, interactions, and trends
- **Moderation Reports**: Review flagging patterns and moderator performance
- **Custom Dashboards**: Create tailored views for your specific needs

### **Moderation Activity Reports**

Comprehensive audit trail system that tracks all moderation activities across your social.plus applications, providing transparency and accountability for content moderation decisions.

#### **What's Tracked**

The moderation activity report captures all key moderation events:

- **Content Actions**: Post/comment flagging, deletion, and flag clearing
- **User Management**: Global bans, community bans, and user removals
- **Message Moderation**: Chat message flags, deletions, and AI detection events
- **Community Events**: Member bans, removals, and role changes
- **AI Detection**: Automated AI moderation, topic detection, and PII identification

#### **Report Data Structure**

Each report entry includes comprehensive metadata in a structured format:

| Field | Description |
|-------|-------------|
| `network_id` | Your network identifier |
| `user_id` | Public ID of the content creator |
| `event` | Type of moderation event |
| `note` | Reason or note for the action |
| `content_type` | Type of content moderated |
| `content_public_id` | Unique identifier for the content |
| `target_type` | Context type where action occurred |
| `target_id` | Target identifier |
| `is_ai` | Whether action was automated |
| `created_at` | When the event occurred |
| `updated_at` | When the record was last updated |
| `metadata` | Additional context and actor information |

#### **Key Fields**

<AccordionGroup>
  <Accordion title="Event Types">
    - `post.flagged`, `post.deleted`, `post.flagsCleared`
    - `comment.flagged`, `comment.deleted`, `comment.flagsCleared`
    - `v5.message.flagged`, `v5.message.deleted`, `v5.message.flagCleared`
    - `community.userBanned`, `community.userRemoved`
    - `user.didGlobalBan`, `user.flagCleared`
  </Accordion>
  <Accordion title="Content Information">
    - `content_type`: Type of content (post, comment, message, community, user)
    - `content_public_id`: Unique identifier for the content
    - `target_type` & `target_id`: Context where the action occurred
  </Accordion>
  <Accordion title="Actor & Timing">
    - `user_id`: Public ID of the content creator
    - `is_ai`: Whether the action was performed by AI automation
    - `created_at` & `updated_at`: Timestamps for audit trail
    - `metadata`: Additional context including actor information
  </Accordion>
</AccordionGroup>

#### **Enabling Reports**

<Warning>
**Feature Activation Required**: Moderation activity reports must be enabled before data collection begins. Historical data before activation is not available.
</Warning>

To enable moderation activity reports for your network:

<Tabs>
  <Tab title="API Method">
    Use the [Update Moderation Network Settings API](/api-reference/network-setting/update-moderation-configuration) to enable reporting:

    ```json
    {
      "isReportEnabled": true
    }
    ```
  </Tab>
  <Tab title="Support Contact">
    Contact our support team at [support@social.plus](mailto:support@social.plus) to enable moderation activity reports for your network.

    Include your network ID and specify that you need moderation reporting activated.
  </Tab>
</Tabs>

## Common Workflows

<AccordionGroup>
  <Accordion title="Content Moderation Workflow">
    1. **Configure Rules**: Set up automated content filtering and AI moderation
    2. **Review Queue**: Process flagged content through moderation dashboard
    3. **Take Actions**: Approve, remove, or escalate content based on guidelines
    4. **Monitor Results**: Track moderation effectiveness and adjust policies
  </Accordion>
  <Accordion title="User Management Workflow">
    1. **Assign Roles**: Set up moderators and assign appropriate permissions
    2. **Monitor Behavior**: Track user activity and identify problematic patterns
    3. **Enforce Policies**: Apply warnings, temporary restrictions, or bans
    4. **Handle Appeals**: Process user appeals and review moderation decisions
  </Accordion>
  <Accordion title="Moderation Activity Report Workflow">
    1. **Enable Reporting**: Activate moderation activity reports via API or support request
    2. **Monitor Activities**: Track all moderation actions in real-time as they occur
    3. **Generate Reports**: Download CSV reports for analysis and compliance auditing
    4. **Analyze Trends**: Review patterns in moderation actions and moderator performance
  </Accordion>
  <Accordion title="Analytics Review Workflow">
    1. **Daily Monitoring**: Check key metrics for user activity and engagement
    2. **Trend Analysis**: Identify patterns in content performance and user behavior
    3. **Report Generation**: Create regular reports for stakeholders
    4. **Strategy Adjustment**: Modify features and policies based on data insights
  </Accordion>
</AccordionGroup>

## Integration with SDK

The analytics and moderation tools work seamlessly with your social.plus SDK implementation:

- **Console Settings** automatically apply to SDK behavior
- **API Actions** can be triggered by SDK events and webhooks
- **Moderation Rules** configured in console affect SDK content validation
- **Analytics Data** reflects all SDK-driven user activities

## Best Practices

<Warning>
  **Privacy & Compliance**: Always ensure your moderation and analytics practices comply with relevant privacy laws and platform policies. Maintain transparency with users about data collection and moderation actions.
</Warning>

- **Clear Policies**: Establish and communicate clear community guidelines
- **Balanced Automation**: Combine AI tools with human judgment for best results
- **Regular Review**: Continuously evaluate and improve your moderation processes
- **Data-Driven Decisions**: Use analytics to inform feature and policy changes
- **User Feedback**: Maintain channels for user feedback on moderation decisions
