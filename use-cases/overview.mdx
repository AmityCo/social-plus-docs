# Use Cases

The other tabs in this documentation are excellent **reference material** — deep, precise, and comprehensive. This tab is different. It answers the question: _"I want to build X. Where do I start and how do I connect everything?"_

Each guide walks through a complete social feature end-to-end, showing you:

- The **SDK implementation** — wire up the underlying APIs with full control
- How the feature connects to **Analytics & Moderation**, **webhooks**, and **admin tooling**
- Links to the **SDK reference pages** for real, copy-paste-ready code examples

<Info>
All guides assume you have already installed the SDK and completed authentication. If you haven't, start with [SDK Getting Started](/social-plus-sdk/getting-started/overview).
</Info>

## How to Use These Guides

Each guide follows the same structure so you can navigate quickly:

<Steps>
  <Step title="What You'll Build">
    A quick summary with an architecture diagram so you know exactly what the guide covers before you read it.
  </Step>
  <Step title="Quick Start">
    A representative code snippet showing the core operation, pulled directly from the SDK reference.
  </Step>
  <Step title="Step-by-Step Implementation">
    Each step links to the SDK reference page with real, copy-paste-ready code for all platforms — iOS, Android, Web (TypeScript), and Flutter.
  </Step>
  <Step title="Connect to Moderation & Analytics">
    How the feature plugs into the Admin Console, webhooks, and reporting.
  </Step>
  <Step title="Best Practices & Next Steps">
    Performance tips, UX patterns, and links to the next guide in the journey.
  </Step>
</Steps>
